System Flow

Treasury Agent Autonomous buyer starts from a target RWA opportunity and requested allocation.
MCP Discovery Paid tools are listed before the agent calls risk, KYB, liquidity, or covenant providers.
HTTP 402 Providers return x402-Casper requirements before data access.
Signed Payment Ed25519 authorizations include nonce, expiry, asset id, tool, and amount.
Paid Intelligence Four RWA signals feed the allocation policy.
Policy Decision Risk gates and caps produce approve, limit, or reject outcomes.
Verifier Hashes, signatures, revenue, and receipt metadata recompute independently.
Casper Anchor Final receipt and settlement anchors are public on Casper testnet.
Judge readout

This is a full agent-commerce loop, not a standalone chat demo: discovery, payment, paid intelligence, policy, verifier, provider revenue, settlement anchors, and final Casper receipt.

Threat Model

Replay or fake payment Nonce replay returns HTTP 402; signed authorization hashes are verified in the judge proof pack.
Local-only settlement claim Four x402 settlement-anchor transactions are linked to public CSPR.live transaction pages.
Unsupported allocation Risk, KYB, liquidity, and covenant reports are paid inputs to the deterministic policy.
Screenshot-only proof JSON artifacts, SHA-256 manifest, and verifier output let judges recompute the evidence.
Secret leakage Public artifacts and final zip are scanned for private-key patterns before publishing.
One happy path only The scenario matrix replays the same paid stack across treasury, invoice-finance, and trade-credit assets.
